# Mastering Next.js: A Comprehensive Guide

## Introduction to Next.js

Next.js is a powerful React framework that enables developers to build fast, user-friendly web applications with ease, server-side rendering (SSR), static site generation (SSG), and incremental static regeneration (ISR). Created by Vercel, Next.js has become the go-to choice for modern web development due to its performance optimizations, developer experience, and built-in features.

As of 2025, Next.js 15 introduces the React Compiler, improved caching semantics, and enhanced partial prerendering, making it more efficient than ever. This 1500-word guide covers everything from setup to advanced patterns.

---

## Getting Started with Next.js

### Installation

To create a new Next.js app, run:

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

This sets up a development server at `http://localhost:3000`.

### Project Structure

```
my-app/
├── app/                # App Router (recommended)
├── pages/              # Pages Router (legacy)
├── public/             # Static assets
├── styles/             # CSS files
├── components/         # Reusable components
├── lib/                # Utility functions
├── next.config.js      # Configuration
└── package.json
```

Next.js 13+ recommends the **App Router** (`app/`) over the legacy Pages Router (`pages/`).

## The App Router (app/)

The `app/` directory introduces a file-system-based router with enhanced features.

### Route Segments

Each folder in `app/` represents a route segment:

```
app/
├── layout.tsx          # Shared layout
├── page.tsx            # Home page (/)
├── about/
│   └── page.tsx        # /about
├── dashboard/
│   ├── layout.tsx      # Nested layout
│   └── page.tsx        # /dashboard
```

### Layouts

Layouts wrap pages and maintain state on navigation:

```tsx
// app/layout.tsx
export default function RootLayout({
  children,
}: {
  children: React.ReactNode;

  return (
    <html lang="en">
      <body>
        <header>My App</header>
        {children}
        <footer>© 2025</footer>
      </body>
    </html>
  );
}
```

### Pages

## Data Fetching Strategies

Next.js supports multiple rendering strategies.

### Server Components (Default)

All components in `app/` are Server Components by default:

```tsx
// app/users/page.tsx
async function getUsers() {
  const res = await fetch('https://api.example.com/users');
  return res.json();
}

export default async function UsersPage() {
  const users = await getUsers();
  return (
    <ul>
      {users.map((user: any) => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
```

### Client Components

Use `'use client'` for interactivity:

```tsx
'use client';

import { useState } from 'react';

export default function Counter() {
  const [count, setCount] = useState(0);
  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}
```

### Static Rendering (SSG)

```tsx
// app/blog/[slug]/page.tsx
export async function generateStaticParams() {
  return [{ slug: 'post-1' }, { slug: 'post-2' }];
}

export default async function BlogPost({ params }: { params: { slug: string } }) {
  const post = await fetch(`https://api.example.com/posts/${params.slug}`).then(res => res.json());
  return <article>{post.content}</article>;
}
```

### Server-Side Rendering (SSR)

```tsx
export const dynamic = 'force-dynamic';

export default async function TimePage() {
  const time = new Date().toLocaleTimeString();
  return <p>Current time: {time}</p>;
}
```

### Incremental Static Regeneration (ISR)

```tsx
export const revalidate = 60; // Revalidate every 60 seconds

export default async function ISRPage() {
  const data = await fetch('https://api.example.com/data', { next: { revalidate: 60 } });
  return <div>{data.message}</div>;
}
```

## API Routes

Create backend endpoints in `app/api/`:

```tsx
// app/api/hello/route.ts
export async function GET() {
  return Response.json({ message: 'Hello from API!' });
}

export async function POST(request: Request) {
  const body = await request.json();
  return Response.json({ received: body });
}
```

Access at `/api/hello`.

## Middleware

Run code before a request completes:

```ts
// middleware.ts
import { NextResponse } from 'next/server';
import type { NextRequest } from 'next/server';

export function middleware(request: NextRequest) {
  if (request.nextUrl.pathname.startsWith('/admin')) {
    if (!request.cookies.has('auth')) {
      return NextResponse.redirect(new URL('/login', request.url));
    }
  }
  return NextResponse.next();
}

export const config = {
  matcher: '/admin/:path*',
};
```

### Caching

```ts
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  experimental: {
    staleTimes: {
      dynamic: 30,
      static: 180,
    },
  },
};

export default nextConfig;
```
